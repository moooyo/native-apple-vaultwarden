# KeyVault Implementation Plan (M1 · Plan 3/N)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Checkbox steps.

**Goal:** Build `KeyVault` — a Swift 6 `actor` that owns the in-memory key hierarchy: it performs the **PBKDF2 master-password unlock** (derive → HKDF-stretch → decrypt the protected user key), holds the unwrapped `UserKey` in memory only, vends decryption of `EncString`s (with optional per-cipher keys), and **zeroizes/locks** on demand. This is the decryption heart that ties CryptoCore into "unlock and read a real vault item."

**Architecture:** A SwiftPM library target `KeyVault` depending ONLY on `CryptoCore` (per the spec's L1 layering — orchestration over VaultModels lives later in VaultRepository). `actor KeyVault` serializes all key access; the user key never leaves the actor as raw bytes except through explicit decrypt calls. No persistence, no Keychain (that's KeychainBridge, a separate Xcode-gated plan). Tests use the established CLT executable-target convention (`swift run KeyVaultTests`); since the type is an `actor`, the runner is `async`.

**Tech Stack:** Swift 6 (`swift-tools-version: 6.2`), CryptoCore. Baseline iOS/macOS 26.

> **⚠️ TESTING CONVENTION** (same as Plans 1–2): no XCTest; `.executableTarget` `KeyVaultTests` via `swift run KeyVaultTests`; copy the `TestRunner` harness. **Because KeyVault is an `actor`, make `runAllTests()` `async` and `main.swift` do `let failures = await runAllTests()`.** TDD: add check → fail → implement → pass → commit.

**Public API (lock this surface):**
```swift
public actor KeyVault {
    public init()
    public var isUnlocked: Bool { get }

    /// Real unlock: PBKDF2(masterpw) -> stretch -> decrypt protectedUserKey (a type-2 EncString
    /// wrapping the 64-byte user key) -> hold UserKey. PBKDF2 only (D6).
    public func unlock(password: String, email: String, iterations: Int, protectedUserKey: EncString) throws

    /// Direct unlock when the 64-byte user key is already recovered (e.g. via biometric path later).
    public func unlock(userKey: SymmetricCryptoKey)

    public func lock()

    /// Decrypt an EncString with the user key (throws .locked if not unlocked).
    public func decrypt(_ encString: EncString) throws -> Data
    public func decryptString(_ encString: EncString) throws -> String

    /// Decrypt the per-cipher key (a type-2 EncString wrapping 64 bytes) into a usable key.
    public func cipherKey(fromProtected protectedKey: EncString) throws -> SymmetricCryptoKey

    /// Decrypt a field using a per-cipher key if provided, else the user key.
    public func decrypt(_ encString: EncString, cipherKey: SymmetricCryptoKey?) throws -> Data
}
```

```swift
import Foundation
import CryptoCore

public actor KeyVault {
    private var userKey: SymmetricCryptoKey?

    public init() {}
    public var isUnlocked: Bool { userKey != nil }

    public func unlock(userKey: SymmetricCryptoKey) { self.userKey = userKey }
    public func lock() { userKey = nil }   // value type drops; no long-lived SecureBytes held here

    public func decrypt(_ encString: EncString) throws -> Data {
        guard let userKey else { throw KeyVaultError.locked }
        return try SymmetricCrypto.decrypt(encString, using: userKey)
    }
    public func decryptString(_ encString: EncString) throws -> String {
        let data = try decrypt(encString)
        guard let s = String(data: data, encoding: .utf8) else { throw CryptoError.decryptionFailed }
        return s
    }
}
```
- [ ] **Step 4: Run** → pass. **Step 5: Commit** `feat(keyvault): add actor with direct unlock, decrypt, lock`.

  - Wrong password (`"wrong"`) → `unlock` throws `KeyVaultError.unlockFailed` (the HMAC check inside `SymmetricCrypto.decrypt` fails → mapped).
  - A protectedUserKey that decrypts to ≠64 bytes → `KeyVaultError.invalidUserKey`.
- [ ] **Step 2: Run** → FAIL.
- [ ] **Step 3: Implement**:
```swift
extension KeyVault {
    public func unlock(password: String, email: String, iterations: Int,
                       protectedUserKey: EncString) throws {
        let masterKey = try KDF.deriveMasterKey(password: password, email: email, iterations: iterations)
        let stretched = KeyStretch.stretchMasterKey(masterKey)
        let raw: Data
        do { raw = try SymmetricCrypto.decrypt(protectedUserKey, using: stretched) }
        catch { throw KeyVaultError.unlockFailed }
        guard raw.count == 64 else { throw KeyVaultError.invalidUserKey }
        self.userKey = try SymmetricCryptoKey(combined: raw)
    }
}
```
- [ ] **Step 4: Run** → pass. **Step 5: Commit** `feat(keyvault): add PBKDF2 master-password unlock`.

### Task 4: Per-cipher key decryption

**Files:** extend `Sources/KeyVault/KeyVault.swift`; extend `Checks_Decrypt.swift`.

- [ ] **Step 1: Failing checks** —
  - `cipherKey(fromProtected:)`: encrypt a 64-byte cipher key under the user key → protectedKey EncString; `await vault.cipherKey(fromProtected:)` returns a `SymmetricCryptoKey` equal to the original; locked → throws `.locked`; a protected key decrypting to ≠64 bytes → `.invalidUserKey` (reuse) or a dedicated error.
  - `decrypt(_:cipherKey:)`: a field encrypted under the per-cipher key decrypts when that cipherKey is passed; the same field fails (macMismatch) if the user key is used instead (cipherKey: nil) — proving the two-level path.
- [ ] **Step 2: Run** → FAIL.
- [ ] **Step 3: Implement**:
```swift
extension KeyVault {
    public func cipherKey(fromProtected protectedKey: EncString) throws -> SymmetricCryptoKey {
        guard let userKey else { throw KeyVaultError.locked }
        let raw = try SymmetricCrypto.decrypt(protectedKey, using: userKey)
        guard raw.count == 64 else { throw KeyVaultError.invalidUserKey }
        return try SymmetricCryptoKey(combined: raw)
    }
    public func decrypt(_ encString: EncString, cipherKey: SymmetricCryptoKey?) throws -> Data {
        if let cipherKey { return try SymmetricCrypto.decrypt(encString, using: cipherKey) }
        return try decrypt(encString)
    }
}
```
- [ ] **Step 4: Run** → pass. **Step 5: Commit** `feat(keyvault): add per-cipher key decryption`.
